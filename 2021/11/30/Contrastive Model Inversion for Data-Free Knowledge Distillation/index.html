Constrastive Model Inversion for Data-Free Knowledge Distillation

2021-11-30 机器学习 无数据学习 0 评论 字数统计: 3.6k(字) 阅读时长: 15(分)

主要问题：

现有的KD方法在很大程度上依赖于大量的训练数据，将知识从预先训练好的教师模型转移到学生身上。然而，在许多情况下，由于隐私或传输的原因，训练数据不会与预先训练好的模型一起发布，这使得这些方法无法适用。因此，无数据KD被提出来以解决这个问题。

⽆数据KD的关键步骤是模型反演，其目标是从预先训练过的教师模型中恢复训练数据。现有的零样本反演⽅法通过拟合先验分布来独⽴地优化每个实例，然⽽这样通常会出现模式崩溃问题，即合成的实例之间⾼度相似，因此对知识蒸馏等下游任务的效率有限。

主要思路：

这篇⽂章提出了对比模型反转（Contrastive Model Inversion），其中数据多样性被明确地建模为⼀ 个可优化的⽬标，以缓解模式崩溃的问题

实验表明在相同数据量的约束下，较⾼的数据多样性通常表明有更强的实例辨别能⼒，为此作者在CMI中引⼊了⼀个对比性的学习⽬标，它⿎励将合成的实例与前⼏批中已经合成的实例区分开来，其中正图像对包含相同数据实例的裁剪图像和整个图像，⽽负图像对包含两个不同的数据实例，通过⿎励正图像对相互接近，将负图像对分开分离，CMI显著提⾼了图像的多样性和真实性，从而提高了⽆数据KD的性能。

主要贡献

  • 提出了数据多样性的定义，这使我们能够将多样性明确地纳入优化目标，以提高生成数据的多样性。
  • 提出了一种新的对比性模型反转方法来处理无数据KD中的模式崩溃，同时强制生成的数据分布更加模仿真实数据。
  • 进行了广泛的实验来验证CMI相对于现有技术水平的优越性。

方法

基本符号

模型反演是⽆数据知识蒸馏的⼀个重要步骤，旨在从预先训练好的教师模型ft(x;θt)f_t(x;\theta_t)ft​(x;θt​)中恢复训练数据X′X'X′，作为不可访问的原始数据XXX的替代

在这部分中，我们将讨论三种典型的反演技术：

BN regularization：

该⽅法最初在[Yinetal.2020]中引⼊，即通过对⽹络中的特征进行高斯假设来规范XXX的分布

正则化通常表示为特征统计量N(μl(x),σl2(x))\mathcal{N}\left(\mu_{l}(x), \sigma_{l}^{2}(x)\right)N(μl​(x),σl2​(x))和Batch归⼀化统计量N(μl,σl2)\mathcal{N}\left(\mu_{l}, \sigma_{l}^{2}\right)N(μl​,σl2​)之间的差异， 即：

Lbn(x)=∑lD(N(μl(x),σl2(x)),N(μl,σl2))\mathcal{L}_{b n}(x)=\sum_{l} D\left(\mathcal{N}\left(\mu l(x), \sigma_{l}^{2}(x)\right), \mathcal{N}\left(\mu_{l}, \sigma_{l}^{2}\right)\right) Lbn​(x)=l∑​D(N(μl(x),σl2​(x)),N(μl​,σl2​))

Class prior：

该⽅法通常被引入用于类条件生成，即对x∈X′x\in X'x∈X′上的网络预测做出one-hot的假设[Chen等⼈,2019]，给定⼀个预定义的类别ccc，它⿎励最小化交叉熵损失：

Lcls(x)=CE(ft(x),c)\mathcal{L}_{c l s}(x)=C E\left(f_{t}(x), c\right) Lcls​(x)=CE(ft​(x),c)

Adversarial Distillation：

该方法使得xxx在教师输出ft(x;θt)f_t(x;\theta_t)ft​(x;θt​)和学生输出fs(x;θs)f_s(x;\theta_s)fs​(x;θs​)之间产⽣差异来进⾏对抗性训练，即最大化 KL 散度：

Ladv (x)=−KL(ft(x)/τ∥fs(x)/τ)\mathcal{L}_{\text {adv }}(x)=-K L\left(f_{t}(x) / \tau \| f_{s}(x) / \tau\right) Ladv ​(x)=−KL(ft​(x)/τ∥fs​(x)/τ)

将上述三种⽅案进⾏加权就可以得到⼀个同⼀的反演框架：

Linv=α⋅Lbn(x)+β⋅Lcls(x)+γ⋅Ladv(x)\mathcal{L}_{\mathrm{inv}}=\alpha \cdot \mathcal{L}_{b n}(x)+\beta \cdot \mathcal{L}_{c l s}(x)+\gamma \cdot \mathcal{L}_{a d v}(x) Linv​=α⋅Lbn​(x)+β⋅Lcls​(x)+γ⋅Ladv​(x)

但是由于在这个框架中没有明确地施加多样性约束，传统的反演⽅法可能往往是“惰性的”，并重复合成重复的样本

具体实现

overview:

有了预先训练好的教师模型ft(x;θt)f_t(x;\theta_t)ft​(x;θt​)，对比性模型反演的目标是产生一组具有强烈多样性的x∈X′x\in X'x∈X′，有了它就可以从教师那里提取全面的知识。动机就是：在相同数据量的约束下，更高的多样性通常表示更强的实例可区分性。为此，用实例判别问题对数据多样性进行建模[Wu et al.,2018]，并通过对比性学习构建一个可优化的目标。

数据多样性的定义：

给定⼀组数据X′X'X′，对数据多样性的直观描述将是“数据集中的样本有多大的可区分性”，这揭示了多样性和实例的可区分性之间的正相关，因此如果我们有⼀个特定的度量d(x1.x2)d(x_1.x_2)d(x1​.x2​)来估计⼀个实例对(x1,x2)(x_1,x_2)(x1​,x2​)的可区分性，那么我们就可以为数据多样性给出⼀个明确的定义如下：

Ldiv (X)=Ex1,x2∈X[d(x1,x2)]\mathcal{L}_{\text {div }}(\mathcal{X})=\mathbb{E}_{x_{1}, x_{2} \in \mathcal{X}}\left[d\left(x_{1}, x_{2}\right)\right] Ldiv ​(X)=Ex1​,x2​∈X​[d(x1​,x2​)]

有各种方法来定义metricd(⋅,⋅)d(\cdot, \cdot)d(⋅,⋅)，导致不同的多样性标准。这个距离度量是无界的，我们无法弄清楚它应该有多大才能表明一个好的多样性。在这种情况下，在metric上最大化这种距离度量可能只会导致不正常的结果。因此，需要一个更合适的embedding 空间来构建一个有意义的可区分性度量。

从对比学习中获得的数据多样性：

对比学习最初是为了以自我监督的方式从数据中学习有用的表征，其中通过将每个样本视为一个不同的类别来建立样本级的歧视问题[Wu et al., 2018]。

通过对比学习，⽹络可以学习如何区分不同的实例，这与我们对度量d(.;.)d(.;.)d(.;.)的要求完全⼀致，因此，在此基础上，作者在教师⽹络ftf_tft​上引⼊了另⼀个⽹络hhh作为实例鉴别器，它接受特征ft(x)f_t(x)ft​(x)作为输⼊，并将其投影到⼀个新的嵌⼊空间中

因为教师⽹络是固定的，因此在hhh的新嵌⼊空间中，我们使⽤简单的余弦相似性来描述数据对x1x_1x1​和x2x_2x2​之间的关系如下：

sim⁡(x1,x2,h)=⟨h(x1),h(x2)⟩∥h(x1)∥⋅∥h(x2)∥\operatorname{sim}\left(x_{1}, x_{2}, h\right)=\frac{\left\langle h\left(x_{1}\right), h\left(x_{2}\right)\right\rangle}{\left\|h\left(x_{1}\right)\right\| \cdot\left\|h\left(x_{2}\right)\right\|} sim(x1​,x2​,h)=∥h(x1​)∥⋅∥h(x2​)∥⟨h(x1​),h(x2​)⟩​

然后，可以用对比学习框架[Chen et al., 2020]的形式来表述实例判别问题，每个实例将被随机转化为不同的views ，并应被正确匹配。

对于每个实例x∈X′x\in X'x∈X′，我们通过随机增强构造⼀个正视图x+x^+x+，并将其他实例x−x^-x−视为负实例，对比学习损失形式如下：

Lcr(X,h)=−Exi∈X[log⁡exp⁡(sim⁡(xi,xi+,h)/τ)∑jexp⁡(sim⁡(xi,xj−,h)/τ)]\mathcal{L}_{c r}(\mathcal{X}, h)=-\mathbb{E}_{x_{i} \in \mathcal{X}}\left[\log \frac{\exp \left(\operatorname{sim}\left(x_{i}, x_{i}^{+}, h\right) / \tau\right)}{\sum_{j} \exp \left(\operatorname{sim}\left(x_{i}, x_{j}^{-}, h\right) / \tau\right)}\right] Lcr​(X,h)=−Exi​∈X​[log∑j​exp(sim(xi​,xj−​,h)/τ)exp(sim(xi​,xi+​,h)/τ)​]

其中，τ\tauτ表示温度，鉴别器hhh可以学习如何通过将正对推进并将负对分开来区分不同的样本，这为任何(x1.x2)(x_1.x_2)(x1​.x2​)对提供了一个“对比”度量

为了进一步理解它与数据多样性的关系，让我们定义度量d：

d(x1,x2)=−exp⁡(sim⁡(x1,x2,h)/τ)exp⁡(sim⁡(x1,x1+,h)/τ)d\left(x_{1}, x_{2}\right)=-\frac{\exp \left(\operatorname{sim}\left(x_{1}, x_{2}, h\right) / \tau\right)}{\exp \left(\operatorname{sim}\left(x_{1}, x_{1}+, h\right) / \tau\right)} d(x1​,x2​)=−exp(sim(x1​,x1​+,h)/τ)exp(sim(x1​,x2​,h)/τ)​

将其带入多样性的定义，就可以得到:

Ldiv (X)=−Exi∈XExj∈X[exp⁡(sim⁡(xi,xj−,h)/τ)exp⁡(sim⁡(xi,xi+,h)/τ)]=−1Z(x−)Exi∈X[∑jexp⁡(sim⁡(xi,xj−,h)/τ)exp⁡(sim⁡(xi,xi+,h)/τ)]=1Z(x−)⋅Lcr(X,h)\begin{aligned} \mathcal{L}_{\text {div }}(\mathcal{X}) &=-\mathbb{E}_{x_{i} \in \mathcal{X}} \mathbb{E}_{x_{j} \in \mathcal{X}}\left[\frac{\exp \left(\operatorname{sim}\left(x_{i}, x_{j}^{-}, h\right) / \tau\right)}{\exp \left(\operatorname{sim}\left(x_{i}, x_{i}+, h\right) / \tau\right)}\right] \\ &=-\frac{1}{Z\left(x^{-}\right)} \mathbb{E}_{x_{i} \in \mathcal{X}}\left[\frac{\sum_{j} \exp \left(\operatorname{sim}\left(x_{i}, x_{j}^{-}, h\right) / \tau\right)}{\exp \left(\operatorname{sim}\left(x_{i}, x_{i}^{+}, h\right) / \tau\right)}\right] \\ &=\frac{1}{Z\left(x^{-}\right) \cdot \mathcal{L}_{c r}(\mathcal{X}, h)} \end{aligned} Ldiv ​(X)​=−Exi​∈X​Exj​∈X​[exp(sim(xi​,xi​+,h)/τ)exp(sim(xi​,xj−​,h)/τ)​]=−Z(x−)1​Exi​∈X​[exp(sim(xi​,xi+​,h)/τ)∑j​exp(sim(xi​,xj−​,h)/τ)​]=Z(x−)⋅Lcr​(X,h)1​​

其中，常数Z(x−)Z(x^{-})Z(x−)是指每个实例的负样本的数量，因此我们可以通过最⼩化对比性损失来直接最⼤化多样性 ，因此我们将数据多样性与对比学习目标连接起来，可以直接优化，使数据更加多样化

模型反演：

在上一部分中，我们将数据多样性与对比性学习目标衔接起来，可以直接优化，使数据更加多样化。因此，将对比性学习整合到模型反演中，形成我们最终的算法，即对比性模型反演。

如图所示，作者的⽅法由四个组件组成：生成器g(⋅;θg)g\left(\cdot ; \theta_{g}\right)g(⋅;θg​)、教师⽹络ft(⋅;θt)f_t\left(\cdot ; \theta_{t}\right)ft​(⋅;θt​)、实例鉴别器h(⋅;θh)h\left(\cdot ; \theta_{h}\right)h(⋅;θh​)和内存库BBB

鉴别器是⼀种简单的多层感知[Chenetal.，2020]，它接受倒数第⼆层的表征以及中间特征的global pooling作为输⼊

CMI的核心思想是逐步合成一些新的样本，这些样本可以很容易地与存储库中的历史样本区分开来。因此，模型反转过程是以 "case-by-case "策略处理的，这意味着在每个时间戳T中，生成器将只合成一批数据。具体来说，在时间戳T的开始，我们重新初始化生成器，并迭代优化其latent code z和参数θg用于合成。

与[Yinetal.，2020]中使⽤的独⽴更新不同像素的像素更新策略相比，“case-by-case”⽣成器可以对像素提供更强的正则化，因为它们是由共享的权值θg\theta_gθg​，并且在合成过程中，对合成图像应应用随机增强，产⽣局部视图xxx和全局视图x+x^+x+进行对比学习。注意单批样本将不足以训练鉴别器，因此，我们让存储在内存库中的历史图像也参与了学习过程，这样对比模型反演的目标可以形式化如下：

min⁡θg,z,h[αcr⋅Lcr(g(z;θg)∪B,h)+βinv⋅Linv(g(z;θg))]\min _{\theta_{g}, z, h}\left[\alpha_{c r} \cdot \mathcal{L}_{c r}\left(g\left(z ; \theta_{g}\right) \cup \mathcal{B}, h\right)+\beta_{i n v} \cdot \mathcal{L}_{i n v}\left(g\left(z ; \theta_{g}\right)\right)\right] θg​,z,hmin​[αcr​⋅Lcr​(g(z;θg​)∪B,h)+βinv​⋅Linv​(g(z;θg​))]

其中，Linv\mathcal{L}_{i n v}Linv​是之前提到过的反演准则，仅适⽤于图像的全局视图，而Lcr\mathcal{L}_{c r}Lcr​同时考虑了合成批g(z;θg)g(z ; \theta_{g})g(z;θg​)和l来自BBB的历史数据，其中历史将为当前图像合成提供有用的指导

在对比学习过程中，我们停止全局视图上的梯度，只允许在局部视图上反向传播，该操作可以为局部模式合成提供更清晰的梯度

对比性模型反演的完整算法:

知识蒸馏：

有了inverted数据X，很容易用Kullback-Leibler散度来训练学生。然而，合成也许不是知识转移的最佳方式，其中一些重要的模式被遗漏。对抗性蒸馏法是一种流行的提高学生成绩的技术，它将学生纳入到图像合成中，用公式10使教师和学生之间的散度最大化。然而，大的散度可能并不总是对应于有价值的样本，因为它们可能只是一些异常值。在这项工作中，我们更加关注那些边界样本，并引入决策对抗性损失。

Ld−adv=−KL(ft(x)/τ∥fs(x)/τ)1{arg⁡max⁡ft(x)=arg⁡max⁡fs(x)}\begin{array}{rl} \mathcal{L}_{d-a d v}=-KL\left(f_{t}(x) / \tau \| f_{s}(x) / \tau\right) \\ 1\left\{\arg \max f_{t}(x)=\arg \max f_{s}(x)\right\} \end{array} Ld−adv​=−KL(ft​(x)/τ∥fs​(x)/τ)1{argmaxft​(x)=argmaxfs​(x)}​

函数1 { − } 是一个指标，当教师和学生对x产生相同的预测时，启用对抗和学习，否则禁用。与公式10中的无界损失项不同，我们的决策对抗性损失将使x接近决策边界，这可以提供更多关于教师网络的信息。

实验结果：

可视化了来自DeepInversion（没有对抗性蒸馏的ADI）、DFQ和本文的方法的合成数据。结果显示，本文的方法能够合成更多可信的和多样化的数据。DeepInversion和DFQ都存在模型崩溃的问题。

结论

在这项工作中，提出了一种新的模型反演方法来保证合成数据的多样性，这可以为下游蒸馏任务带来显著的好处。本文进行了大量的实验，展示了方法的有效性。然而，在知识提炼的背景下，模型反演仍然是一个有趣的话题，这可能会提供一些关于理解网络中所学知识以及如何将其转移到其他网络的见解。
